# Game Design Documents

This folder contains all mechanical and systems design documentation.

## Key Documents to Create

- **GDD (Game Design Document)** — The master reference for all game systems
- **Combat System** — Turn-based combat mechanics, abilities, status effects
- **Progression System** — Leveling, "insight" XP, skill trees
- **Item System** — Equipment, consumables, quest items, joke items
- **Difficulty & Balance** — Zone scaling, enemy stats, economy balance
- **UI/UX Design** — Menu flows, HUD layout, interaction patterns

## Design Principles

1. **Writing first** — Every mechanic should serve a joke or narrative beat
2. **Accessible complexity** — Easy to pick up, depth comes from humor not min-maxing
3. **Rewarding exploration** — Hidden jokes, Easter eggs, and absurd item descriptions everywhere
4. **Subvert RPG tropes** — Use the player's genre expectations against them
